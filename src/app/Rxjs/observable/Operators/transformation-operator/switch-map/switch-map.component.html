Switch Map

-- Switch Map is a flat Map
-- It is used to access the observable data that is return from another observable.(Nested observable)
--Main difference is cancelling effect.
--On each emission the previous inner observable is cancelled and the new observable is subscribed.

Ex-01

Map

--Using Map -- we need to subscribe twice (observables of observables )or(Nested observable) to get the values
-- If subscribe once it will give object(only observable data)


Ex-02

Map + SwitchAll

SwitchAll --Flat observable of observables. Once the previous value emited the it will cancel and emit the new value


Ex-03

SwitchMap = Map+SwitchAll


Exhaust Map

-- Map to inner observable, ignore other values until that observable completes.
--do not distrube mode
--if it is working on the inner observable,ignore other values untill get the response of the first one
--it is used to avoid the accidental multiple request to the server
